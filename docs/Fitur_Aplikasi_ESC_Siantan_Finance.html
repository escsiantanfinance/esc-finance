Ringkasan & Manfaat Utama

Sistem Akuntansi & Bendahara Gereja adalah platform digital yang memodernisasi pencatatan keuangan pasca-ibadah — menggantikan formulir kertas dan input manual ke spreadsheet yang rawan salah ketik, kehilangan data, dan keterlambatan laporan.

Kunci & Sinkronisasi

1. Pembukaan Sesi

Anggota Finance membuka aplikasi, memilih jenis ibadah (Ibadah Raya, Pemuda, Sekolah Minggu), tanggal, dan jam. Semua data persembahan terikat rapi pada sesi ini.

2. Input Denominasi & Kategori

Tim memasukkan jumlah lembar uang per pecahan (mis. 62 lembar Rp100.000) — sistem menghitung totalnya otomatis. Lalu mengelompokkan dana ke kategori: Perpuluhan, Janji Iman, Persembahan Umum, dll.

3. Balancing Sistem

Sistem membandingkan total fisik uang dengan total kategori. Hasilnya ditampilkan sebagai indikator:

● MATCH (Hijau)   total cocok, boleh lanjut    ● MISMATCH (Merah)   ada selisih, harus dicek ulang

4. Otorisasi & Tanda Tangan

Jika sudah MATCH, kanvas tanda tangan aktif. Gembala dan Saksi membubuhkan tanda tangan digital langsung di layar sebagai bukti sah.

5. Kunci Data & Sinkronisasi

Setelah ditandatangani, data otomatis terkunci (tidak bisa diubah di aplikasi) dan terkirim ke Web Admin untuk memperbarui pos-pos kas terkait secara real-time.

Pengaman: Tombol "Simpan & Kunci" hanya aktif jika status data sudah seimbang (balanced). Ini mencegah penyimpanan data yang tidak akurat.

Fitur Aplikasi Mobile (Tim Finance)

Difokuskan pada entri data yang cepat, aman, dan meminimalkan kesalahan ketik di lapangan.

FiturDeskripsi & Cara Kerja
Manajemen Sesi Ibadah Memilih jenis ibadah, tanggal, dan jam sesi untuk melokalisasi data persembahan. Setiap transaksi terikat pada metadata ibadah yang spesifik.
Kalkulator Denominasi Input jumlah lembar uang per pecahan (Rp100.000 sampai Rp1.000). Sistem menghitung subtotal & total fisik otomatis — menghapus risiko salah hitung manual.
Form Kategori Persembahan Input nominal per kategori: Perpuluhan, Janji Iman, Kolekte Umum, Persembahan Khusus, dll.
Validasi MATCH / MISMATCH Indikator hijau/merah membandingkan total fisik vs total kategori secara real-time. Mencegah data yang tidak seimbang tersimpan.
Tanda Tangan Digital (E-Signature) Kanvas tanda tangan untuk Gembala & Saksi. Aktif hanya bila status sudah balanced. Tanda tangan tersimpan sebagai bukti otorisasi.
Penyimpanan Lokal Sementara Setiap ketikan otomatis disimpan ke memori HP. Jika aplikasi tertutup atau baterai habis, data input tidak hilang dan bisa dilanjutkan.
Dashboard & Saldo Menampilkan ringkasan saldo kas dan akses cepat ke fungsi utama.
Pengajuan Pengeluaran Mengajukan permintaan pengeluaran beserta unggah foto bukti/nota untuk disetujui Bendahara.
Anti-hilang data: kombinasi penyimpanan lokal + sinkronisasi cloud memastikan tidak ada data yang hilang sebelum proses unggah selesai sepenuhnya.

Fitur Web Dashboard (Admin & Akuntansi)

Kendali penuh atas tata kelola keuangan, struktur akuntansi, dan pengeluaran kas gereja.

A. Modul Akuntansi Standar & Multi-Kas

FiturDeskripsi
Bagan Akun (Chart of Accounts) Struktur kode akun akuntansi nirlaba yang memisahkan Aset, Kewajiban, Ekuitas (Dana Terikat & Tidak Terikat), Pendapatan, dan Beban.
Manajemen Multi-Kas Pengaturan pos kas/rekening spesifik (Kas Operasional, Kas Pembangunan, Kas Diakonia, dll). Setiap pengeluaran wajib memilih pos kas asal.
Jurnal Umum & Penyesuaian Pencatatan transaksi non-persembahan secara manual: biaya admin bank, penyusutan aset, atau koreksi pembukuan — dengan validasi debit = kredit.
Manajemen Pengeluaran Input pengeluaran disertai unggah bukti nota. Alur persetujuan: diajukan → disetujui/ditolak oleh Bendahara.
Anggaran (Budgeting) Penetapan pagu anggaran per divisi/kategori untuk mengontrol agar pengeluaran tidak melebihi rencana bulanan/tahunan.

B. Pemantauan & Otorisasi

Tinjau Sesi Ibadah

Melihat rincian setiap sesi: denominasi, kategori, status MATCH/MISMATCH, dan gambar tanda tangan Gembala & Saksi.

Persetujuan Pengeluaran

Menyetujui atau menolak pengajuan pengeluaran dari Tim Finance lengkap dengan catatan.

Export Excel

Mengekspor data persembahan, pengeluaran, anggaran, dan laporan ke file Excel (.xlsx) untuk analisis mandiri.

Dashboard Ringkas

Ringkasan saldo total, pemasukan & pengeluaran bulanan, grafik tren, dan transaksi terbaru.

Modul Laporan Keuangan

Tiga laporan keuangan standar akuntansi nirlaba, diturunkan otomatis dari satu sumber data (Jurnal Umum) sehingga selalu konsisten.

📈 1. Laporan Aktivitas (Income Statement)

Menampilkan total pendapatan persembahan berdasarkan kategori dikurangi total pengeluaran beban operasional, menghasilkan surplus/defisit periode.

⚖️ 2. Laporan Posisi Keuangan (Neraca)

Menyajikan saldo riil seluruh akun kas/bank, piutang janji iman, aset tetap, serta posisi ekuitas dana gereja (terikat & tidak terikat).

💧 3. Laporan Arus Kas (Cash Flow)

Melacak likuiditas — mutasi kas masuk & keluar yang dikelompokkan berdasarkan aktivitas operasional, investasi, dan pendanaan.

Pelaporan Tambahan

  • Anggaran vs Realisasi — perbandingan pagu anggaran dengan pengeluaran aktual per kategori/bulan.
  • Ringkasan Bulanan — total persembahan dan jumlah transaksi per bulan.
  • Komposisi Pengeluaran — grafik proporsi pengeluaran per kategori.
  • Export ke Excel — setiap laporan dapat diunduh dalam format .xlsx.
Keunggulan teknis: karena semua laporan bersumber dari Jurnal Umum yang sama, angka antar-laporan dijamin konsisten — tidak ada selisih akibat perhitungan terpisah.

Modul Pencadangan & Pemulihan Data

Menjamin keamanan data keuangan tingkat tinggi dari risiko kegagalan sistem maupun kendala operasional.

🕑 Pencadangan Otomatis

Sistem mencadangkan database penuh setiap hari pada jam non-aktif (02:00 WIB). File dienkripsi dan disimpan terpisah dari database utama.

Retensi: 30 cadangan harian + 12 cadangan bulanan.

⬇️ Pencadangan Manual Instan

Tombol "Picu Pencadangan Data" untuk mengunduh salinan kapan saja, dikompresi dalam satu file ZIP.

Format: Excel (.xlsx) untuk dibuka langsung + format sistem untuk pemulihan.

Log Cadangan

Halaman khusus meninjau riwayat backup: tanggal, tipe (Otomatis/Manual), ukuran file, status (Sukses/Gagal), dan pelaksana (Sistem/Nama Admin).

Pemulihan Data (Restore)

Pemulihan dilakukan dengan mengunggah file Excel hasil backup — mudah digunakan dan aman, tanpa perlu keahlian teknis. Sistem menampilkan pratinjau jumlah data sebelum konfirmasi.

Pengamanan ketat: hak akses pemulihan data hanya dapat dieksekusi oleh Super Admin / Gembala Senior dengan otentikasi berlapis.

Keamanan & Teknologi

🔐 Keamanan Data

Keamanan berlapis tingkat baris (Row Level Security), enkripsi file cadangan, dan otentikasi berlapis untuk aksi sensitif.

✍️ Bukti Otorisasi

Tanda tangan digital + penguncian data otomatis memastikan setiap transaksi tervalidasi dan tidak dapat dimanipulasi.

☁️ Infrastruktur Cloud

Supabase (PostgreSQL) — database relasional skala besar dengan sinkronisasi real-time.

📱 Lintas Platform

Aplikasi mobile (Flutter) & web dashboard (Next.js) dapat diakses dari perangkat apa pun.
